## Prerequisites

- A supported platform (macOS or Linux; Windows via WSL2).
- Claude Code or Codex CLI installed and authenticated.
- Go 1.25+ if building from source.

## 1. Install

<CodeGroup>

```bash Homebrew
brew install ericmacdougall/stoke/stoke
```

```bash curl
curl -fsSL https://raw.githubusercontent.com/ericmacdougall/Stoke/main/install.sh | bash
```

```bash Docker
docker pull ghcr.io/ericmacdougall/stoke:latest
```

```bash Source
go build ./cmd/stoke
sudo mv stoke /usr/local/bin/
```

</CodeGroup>

## 2. Run a single task

```bash
stoke run --task "Add request ID middleware" --dry-run
```

R1 will:

1. **Plan** — propose a deterministic task decomposition.
2. **Execute** — run a single strong implementer in an isolated git worktree.
3. **Verify** — run build, test, lint, and scope gates.
4. **Commit** — merge only if every gate passes and the cross-family reviewer concurs.

Drop `--dry-run` to let R1 write, commit, and merge.

## 3. Run a multi-task plan

```bash
stoke build --plan stoke-plan.json --workers 4 --dry-run
```

Tasks dispatch in GRPW priority order (most downstream work first),
and R1 automatically serializes any tasks whose file scopes conflict.

## 4. Generate a plan from a task description

```bash
stoke plan --task "Add JWT auth" --dry-run
```

R1 will inspect your repository, propose a ranked task tree, and
emit a plan you can edit and re-feed into `stoke build`.

## 5. Free-text entry

```bash
stoke task "Fix the flaky integration test in server/handler"
```

The executor router picks the right backend (Claude Code, Codex,
library fallback) based on task type.
